#### Version 1.2.2-beta (2022-05-01) ####
- Added "Export Annotation Stats..." item to the "File" menu. It exports annotation tracker statistics (like
who and when created/deleted/modified how many annotations).

#### Version 1.2.1-beta (2023-04-07) ####
- Annotation source dialog converted to non-modal window, allowing users to work with annotations on the main screen
without closing this window.
- Added "Real User" field, users using the same system account can enter their real names.
- Multiple entries per user per day are now enabled. The "New" button will create a new entry/comment,
the "Delete" button will delete an entry/comment if it was created by mistake and never been used.
- Any comment created today by the same user can bec edited in the table.

#### Version 1.2.0-beta (2022-10-21) ####
- Added annotation source tracking feature. Annotations are associated with a metadata object containing
information about their origin, whether it is auto-generated by certain method with certain parameters, or
added/modified by a human on certain date. The metadata is displayed in the status line (at bottom right
of the main screen) when hovering mouse over an annotation. In addition, there is a dialog (which is open via
menu View -> Annotation Sources... or F6), which allows to gray out annotations associated with selected sources.
The dialog also has a "Comment" box to add user-defined comments on current edits.
The annotation source metadata is saved to regular *_contours.csv files as a specially formatted comment line,
this way maintaining compatibility with the previous versions, including the C++ (non-ML) one.

#### Version 1.0.8 (2022-04-07) ####

- Experimental option of multiple values for "Levelset iterations" is merged into the main version.
  It is disabled by default, but can be enabled via menu "File" -> "Advanced Options" -> "Multiple levelset iterations".
  When enabled, an additional set of controls is displayed on the main screen, and an option for selecting a range
  of values for "Level-set iterations" segmentation parameter is shown inj the Segmentation dialog.
  If this option is selected, multiple results are produced for the specified range, and a slider on the main screen
  is enabled to allow users to browse across different results. The "Freeze" button can be used to purge all annotation
  sets except the current one. Besically, use the slider to find the "best" result, then press "Freeze" to keep it, and
  remove other results.
- Snapshot dialog: added "Source image" checkbox. If unchecked, the source image is replaced with a solid background
  colored with the "Background" color.
- Hyperlinks in About box.

#### Version 1.0.1 (2020-10-14) ####

- Change mouse cursor to OpenHand when Shift is pressed, and to MoveAll when scrolling the image
  with Shift+LeftButton or MiddleButton.
- The "Segment" dialog displays a list of open images with check boxes, allowing the user to select
  which images to segment (and replace existing contours). Initially only images that don't have any
  contours on them are checked, but there is a "V" header to check/uncheck all.
- A "V" symbol is placed next to image name in the list of open images if there is a matching *_contours.csv
  file in the same directory as the image (*.tif) file. An image without this mark may actually have
  some contours on it, but they are not saved.
- History moved to user's home directory and history entry names (automatic backup) get image file timestamp
  to resolve potential naming conflicts (two different images with the same name will have different
  history entries, so no backup contours are loaded into wrong image).
- Output contours file name changed from *.csv to *_contours.csv, and the format changed to flat list
  X1,Y1,X2,Y2,X3,Y3... The old format "[X1,Y1]","[X2,Y2]","[X3,Y3]"... is still recognized and can be loaded
  manually -- via menu "Open Contours..." or drag-and-drop a group of files containing *.csv only.
- Drag and Drop operations fixed to properly handle *.tif, *.csv, directories or any mixture of them.
  For example, if you have images and saved detections in the same directory, you can drag and drop this
  directory to open all of them at once rather than dragging a large list of individual files.
- Logic handling auto-opening contours changed to pick up history (auto-backup) first, and only if
  such entry does not exist, try to open accompanying manually saved *_contours.csv file. For example, when you
  first open a group of images recently copied together with their *_contours.csv segmentations from
  a different computer, the *_contours.csv-s are automatically loaded as well, and copied to backup (history).
  If you make some changes and close the application without saving (or open another group of images,
  which closes the current one), next time the backup entries will be picked instead,
  so your unsaved changes are not lost. 
- A menu command "Show data file locations" (also available via keyboard shortcut Ctrl+L) opens a box
  showing data file locations for the currently displayed image: the full path to the image *.tif,
  its manual save *.csv file name (grayed if not yet saved), and the auto-backup (history) file.
